# TanStack Query: `useMutation` Update 구현 & 섹션 마무리

## 구현 패턴 — delete와 동일

update mutation도 delete와 완전히 동일한 패턴으로 구현.

```
Posts.jsx
  updateMutation = useMutation({ mutationFn: (postId) => updatePost(postId) })
  → updateMutation을 prop으로 PostDetail에 전달

PostDetail.jsx
  버튼 클릭 → updateMutation.mutate(post.id)
  → isPending / isSuccess / isError 상태로 UI 피드백
```

---

## Posts.jsx — mutation 선언 및 전달

```tsx
// 삭제
const deleteMutation = useMutation({
  mutationFn: (postId) => deletePost(postId),
});

// 업데이트
const updateMutation = useMutation({
  mutationFn: (postId) => updatePost(postId),
});

// 포스트 클릭 시 두 mutation 모두 reset
<li
  onClick={() => {
    deleteMutation.reset();
    updateMutation.reset();
    setSelectedPost(post);
  }}
>
  {post.title}
</li>

// PostDetail에 두 mutation 전달
<PostDetail
  post={selectedPost}
  deleteMutation={deleteMutation}
  updateMutation={updateMutation}
/>
```

## PostDetail.jsx — 상태 표시

```tsx
export function PostDetail({ post, deleteMutation, updateMutation }) {
  return (
    <>
      {/* Delete */}
      <div>
        <button onClick={() => deleteMutation.mutate(post.id)}>Delete</button>
        {deleteMutation.isPending && (
          <p className="loading">Deleting the post</p>
        )}
        {deleteMutation.isError && (
          <p className="error">Error deleting the post</p>
        )}
        {deleteMutation.isSuccess && (
          <p className="success">Post was (not) deleted</p>
        )}
      </div>

      {/* Update */}
      <div>
        <button onClick={() => updateMutation.mutate(post.id)}>
          Update title
        </button>
        {updateMutation.isPending && (
          <p className="loading">Updating the post</p>
        )}
        {updateMutation.isSuccess && <p className="success">Post updated!</p>}
        {updateMutation.isError && (
          <p className="error">Error updating the post</p>
        )}
      </div>
    </>
  );
}
```

## 섹션 전체 구조 정리

```
Posts.jsx (부모)
  ├── useQuery         → 포스트 목록 조회
  ├── useQueryClient   → prefetchQuery로 다음 페이지 미리 fetch
  ├── deleteMutation   → useMutation (delete)
  └── updateMutation   → useMutation (update)
        ↓ props
PostDetail.jsx (자식)
  ├── useQuery         → 댓글 목록 조회
  ├── deleteMutation.mutate(post.id) → Delete 버튼
  └── updateMutation.mutate(post.id) → Update 버튼
```

---

## mutation을 부모(Posts)에서 관리하는 이유 최종 정리

|                 | Posts에서 선언                 | PostDetail에서 선언   |
| --------------- | ------------------------------ | --------------------- |
| 상태 안정성     | ✅ 포스트 바뀌어도 유지        | ❌ 리마운트 시 초기화 |
| 수동 reset 필요 | ✅ 포스트 클릭 시 reset() 필요 | ❌ 자동 초기화        |
| 권장 여부       | ✅ 권장                        | ❌ 비권장             |

## 이번 섹션에서 배운 것

| 개념                        | 핵심                                 |
| --------------------------- | ------------------------------------ |
| `useQuery`                  | 데이터 조회, queryKey + queryFn 필수 |
| `isLoading` vs `isFetching` | 캐시 유무로 구분                     |
| `staleTime` / `gcTime`      | 리패치 주기 / 캐시 보관 기간         |
| `queryKey` 의존성           | 쿼리 함수에서 쓰는 값은 키에 포함    |
| 페이지네이션                | currentPage를 queryKey에 포함        |
| prefetch                    | 다음 페이지를 미리 캐시에 저장       |
| `useMutation`               | CUD 작업, queryKey 불필요, 캐시 없음 |
| `reset()`                   | 포스트 변경 시 mutation 상태 초기화  |
