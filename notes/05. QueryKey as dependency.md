# TanStack Query: Query Key를 의존성 배열로 활용하기

## 문제 상황

```tsx
// ❌ 모든 post의 댓글이 같은 캐시를 공유함
const { data } = useQuery({
  queryKey: ["comments"],
  queryFn: () => fetchComments(post.id),
});
```

어떤 포스트를 클릭해도 항상 첫 번째 포스트의 댓글만 표시됨.  
→ 모든 쿼리가 `"comments"` 라는 동일한 키를 공유하기 때문.

---

## 왜 리패치가 안 될까?

쿼리 키가 이미 캐시에 존재하면, 아래 **트리거**가 발생하지 않는 한 리패치하지 않음.

| 트리거             | 설명                                         |
| ------------------ | -------------------------------------------- |
| 컴포넌트 리마운트  | 쿼리를 사용하는 컴포넌트가 다시 마운트될 때  |
| 윈도우 포커스      | 브라우저 탭이 다시 활성화될 때               |
| 수동 리패치        | `useQuery`가 반환하는 `refetch()` 함수 호출  |
| 자동 리패치        | 지정한 인터벌마다 자동 실행                  |
| 뮤테이션 후 무효화 | 서버 데이터 변경 후 `invalidateQueries` 호출 |

→ 포스트 제목 클릭은 위 트리거에 해당하지 않음. stale 상태여도 리패치 안 됨.

---

## 해결책: Query Key에 의존성 추가

```tsx
// ✅ post.id를 키에 포함 → post마다 별도 캐시 슬롯 생성
const { data } = useQuery({
  queryKey: ["comments", post.id],
  queryFn: () => fetchComments(post.id),
});
```

`post.id`가 바뀌면 React Query는 **완전히 새로운 쿼리**로 인식함.

- 각 post마다 독립적인 `staleTime` 보유
- 각 post마다 독립적인 `gcTime` 보유
- 서로 다른 쿼리로 캐시에 별도 저장

---

## Query Key = 의존성 배열

> **쿼리 함수 안에서 사용하는 값은 반드시 쿼리 키에 포함해야 한다.**

React Query ESLint 플러그인도 이를 경고함:

```
The following dependencies are missing in your queryKey: post.id
```

이를 지키지 않으면 쿼리 함수의 입력값이 바뀌어도 캐시를 재사용해서 **잘못된 데이터를 표시**하게 됨.

---

## inactive 쿼리와 gcTime

포스트를 이동하면 이전 포스트의 댓글 쿼리는 **inactive** 상태로 전환됨.

```
현재 보고 있는 포스트 댓글 → active  (gcTime 타이머 없음)
이전에 봤던 포스트 댓글  → inactive (gcTime 타이머 시작)
```

- inactive 상태가 되는 순간부터 `gcTime` 카운트 시작
- `gcTime` 만료 전에 다시 해당 포스트를 열면 → 캐시 데이터 즉시 표시
- `gcTime` 만료 후 → 캐시에서 완전 삭제, 서버에서 새로 fetch

---

## 정리

|           | 잘못된 방식                     | 올바른 방식               |
| --------- | ------------------------------- | ------------------------- |
| Query Key | `["comments"]`                  | `["comments", post.id]`   |
| 캐시      | 모든 post가 공유                | post마다 독립 슬롯        |
| 동작      | 다른 post 클릭해도 리패치 안 됨 | post 변경 시 새 쿼리 실행 |
