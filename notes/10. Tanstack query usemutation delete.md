# TanStack Query: `useMutation` 실전 구현 (Delete)

## 전체 흐름

```
Posts.jsx
  useMutation 선언 → deleteMutation 객체 생성
  → deleteMutation을 prop으로 PostDetail에 전달

PostDetail.jsx
  삭제 버튼 클릭 → deleteMutation.mutate(post.id) 호출
  → mutationFn 실행 → deletePost(postId) API 호출
```

---

## 왜 `PostDetail`이 아닌 `Posts`에서 선언하나?

```
PostDetail은 selectedPost가 바뀔 때마다 언마운트/리마운트됨
→ 그 안에서 useMutation을 선언하면 mutation 상태가 초기화됨
→ isLoading, isError 등의 상태 추적이 불안정해짐

Posts는 항상 마운트된 상태를 유지함
→ deleteMutation 상태가 안정적으로 유지됨
→ prop으로 PostDetail에 내려주는 방식 사용
```

---

## 코드

### Posts.jsx — useMutation 선언 및 전달

```tsx
import { useMutation } from "@tanstack/react-query";
import { deletePost } from "./api";

const deleteMutation = useMutation({
  mutationFn: (postId) => deletePost(postId),
  // queryKey 없음 — 캐시와 무관한 서버 변경 작업이기 때문
});

// PostDetail에 prop으로 전달
<PostDetail post={selectedPost} deleteMutation={deleteMutation} />;
```

### PostDetail.jsx — mutate 호출

```tsx
export function PostDetail({ post, deleteMutation }) {
  return (
    <>
      <button onClick={() => deleteMutation.mutate(post.id)}>Delete</button>
    </>
  );
}
```

---

## mutate 함수의 인자 전달 흐름

```
deleteMutation.mutate(post.id)
        ↓
mutationFn: (postId) => deletePost(postId)
        ↓
DELETE /posts/:id API 호출
```

`useQuery`의 `queryFn`은 인자를 직접 받을 수 없지만,  
`useMutation`의 `mutationFn`은 **`mutate` 호출 시 넘긴 값을 인자로 받을 수 있음**

---

## useQuery vs useMutation 인자 차이

```tsx
// useQuery — queryFn은 외부 변수를 클로저로 참조
useQuery({
  queryKey: ["comments", post.id],
  queryFn: () => fetchComments(post.id), // post.id를 직접 인자로 못 받음
});

// useMutation — mutationFn은 mutate() 호출 시 인자를 직접 받음
useMutation({
  mutationFn: (postId) => deletePost(postId), // mutate(post.id) → postId로 전달됨
});
```

## 다음 단계

현재는 삭제 버튼을 눌러도 사용자에게 **아무런 피드백이 없음.**  
→ `deleteMutation`이 반환하는 상태값으로 UI 피드백 구현 예정

| 상태                       | 설명         |
| -------------------------- | ------------ |
| `deleteMutation.isPending` | 요청 진행 중 |
| `deleteMutation.isSuccess` | 요청 성공    |
| `deleteMutation.isError`   | 요청 실패    |
