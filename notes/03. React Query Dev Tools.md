# TanStack Query: React Query DevTools

## 개요

개발 중에 앱에 추가할 수 있는 컴포넌트로, **모든 쿼리의 상태를 시각적으로 확인**할 수 있음.  
예상대로 동작하지 않을 때 디버깅에 매우 유용함.

---

## 설치

DevTools는 **별도 패키지**로 분리되어 있어 따로 설치해야 함.

```bash
npm install @tanstack/react-query-devtools
```

> 기존 프로젝트에 이미 설치되어 있다면 생략 가능

---

## 사용법

`App.jsx` 최상단에서 import 후, **`QueryClientProvider` 내부** 어디든 추가 가능.

```tsx
import { ReactQueryDevtools } from "@tanstack/react-query-devtools";

function App() {
  return (
    <QueryClientProvider client={queryClient}>
      <Posts />
      <ReactQueryDevtools /> {/* QueryClientProvider 내부면 어디든 OK */}
    </QueryClientProvider>
  );
}
```

---

## 프로덕션 자동 제외

DevTools는 **기본적으로 프로덕션 빌드에 포함되지 않음.**  
`NODE_ENV` 환경변수를 기준으로 자동 판단함.

| 명령어          | NODE_ENV      | DevTools 포함 여부 |
| --------------- | ------------- | :----------------: |
| `npm run dev`   | `development` |      ✅ 포함       |
| `npm run build` | `production`  |      ❌ 제외       |

## 주요 기능

### 쿼리 목록

- **쿼리 키** 기준으로 쿼리를 목록으로 표시
- 각 쿼리의 **상태** 확인 가능

| 상태       | 설명                       |
| ---------- | -------------------------- |
| `fresh`    | 최신 데이터, 리패치 불필요 |
| `fetching` | 현재 데이터 요청 중        |
| `paused`   | 요청이 일시 중단된 상태    |
| `stale`    | 오래된 데이터, 리패치 필요 |
| `inactive` | 사용 중인 컴포넌트 없음    |

- 마지막 업데이트 **타임스탬프** 확인 가능
- DevTools에서 직접 **Refetch / Invalidate** 실행 가능

### Data Explorer

- 쿼리가 반환한 **실제 데이터** 확인
- 개별 항목 드릴다운, 복사, 삭제 가능

### Query Explorer

- 쿼리 내부 상태 확인 (쿼리 키, 옵션, 가비지 컬렉션 시간 등)

---

## 참고: stale 상태란?

DevTools에서 쿼리를 확인하면 기본적으로 `stale` 상태로 표시됨.  
→ **데이터가 오래되었으니 조건이 되면 다시 fetch하겠다**는 의미  
→ stale time 설정으로 조절 가능 (기본값: `0`)
