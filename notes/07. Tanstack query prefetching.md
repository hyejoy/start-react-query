# TanStack Query: Prefetching

## 개념

사용자가 **필요로 할 것 같은 데이터를 미리 fetch해서 캐시에 저장**하는 기법.  
캐시에 데이터가 있으므로 사용자가 해당 페이지로 이동할 때 로딩 없이 즉시 표시 가능.

> 페이지네이션뿐만 아니라 **사용자가 다음에 방문할 가능성이 높은 모든 데이터**에 활용 가능  
> ex) 통계적으로 특정 탭을 다음에 방문할 확률이 높다면 미리 prefetch

---

## Prefetch된 데이터의 특성

- 캐시에 저장되는 즉시 **stale 상태**로 표시됨 (기본값, 설정 가능)
- 실제로 데이터가 필요해지면 stale이므로 리패치 발생
- 리패치하는 동안 캐시 데이터를 먼저 표시 → 로딩 스피너 없음
- **gcTime이 만료되면** 캐시에서 삭제 → 다시 로딩 스피너 표시될 수 있음

---

## 사용법

### 1. useQueryClient로 queryClient 가져오기

```tsx
import { useQuery, useQueryClient } from "@tanstack/react-query";

const queryClient = useQueryClient();
```

### 2. useEffect로 페이지 변경 시 다음 페이지 prefetch

```tsx
useEffect(() => {
  if (currentPage < maxPostPage) {
    const nextPage = currentPage + 1;
    queryClient.prefetchQuery({
      queryKey: ["posts", nextPage], // useQuery와 동일한 키 형태 사용
      queryFn: () => fetchPosts(nextPage),
    });
  }
}, [currentPage, queryClient]);
```

> ⚠️ **onClick에서 실행하지 않는 이유**  
> state 업데이트는 비동기라서 onClick 시점에 currentPage가 이미 업데이트됐는지 보장할 수 없음  
> → useEffect + currentPage 의존성 배열로 안전하게 처리

---

## 쿼리 키를 동일한 형태로 맞춰야 하는 이유

```tsx
// useQuery
queryKey: ["posts", currentPage];

// prefetchQuery
queryKey: ["posts", nextPage]; // ✅ 동일한 형태
```

React Query는 캐시를 **쿼리 키 기준**으로 찾음.  
키 형태가 다르면 prefetch한 데이터를 찾지 못해서 의미가 없어짐.

---

## 동작 흐름

```
1페이지 진입
→ ["posts", 1] fetch (현재 페이지)
→ ["posts", 2] prefetch (다음 페이지, 백그라운드)

next page 클릭
→ ["posts", 2] 이미 캐시에 있음 → 즉시 표시 ✅ (로딩 없음)
→ ["posts", 3] prefetch (다음 페이지, 백그라운드)

next page 클릭
→ ["posts", 3] 이미 캐시에 있음 → 즉시 표시 ✅
→ ...반복
```

항상 현재 페이지보다 **한 페이지 앞**을 미리 준비해두는 구조.

---

## 전체 코드

```tsx
const maxPostPage = 10;

export function Posts() {
  const [currentPage, setCurrentPage] = useState(1);
  const queryClient = useQueryClient();

  // 다음 페이지 prefetch
  useEffect(() => {
    if (currentPage < maxPostPage) {
      const nextPage = currentPage + 1;
      queryClient.prefetchQuery({
        queryKey: ["posts", nextPage],
        queryFn: () => fetchPosts(nextPage),
      });
    }
  }, [currentPage, queryClient]);

  const { data, isLoading, isError, error } = useQuery({
    queryKey: ["posts", currentPage],
    queryFn: () => fetchPosts(currentPage),
    staleTime: 2000,
  });

  if (isLoading) return <h3>Loading...</h3>;
  if (isError) return <h3>{error.toString()}</h3>;

  return (
    <>
      <ul>
        {data.map((post) => (
          <li key={post.id} onClick={() => setSelectedPost(post)}>
            {post.title}
          </li>
        ))}
      </ul>
      <div className="pages">
        <button
          disabled={currentPage <= 1}
          onClick={() => setCurrentPage((prev) => prev - 1)}
        >
          Previous page
        </button>
        <span>Page {currentPage}</span>
        <button
          disabled={currentPage >= maxPostPage}
          onClick={() => setCurrentPage((prev) => prev + 1)}
        >
          Next page
        </button>
      </div>
    </>
  );
}
```
