# fallback을 쓰는 이유

## 문제 상황

```ts
// useQuery는 처음에 data = undefined 반환
const { data } = useQuery({ queryKey: [...], queryFn: getTreatments })

// Treatments.tsx에서
treatments.map(...)  // ❌ undefined.map() → 에러!
```

fetch가 완료되기 전까지 `data = undefined` 이라서
`.map()`을 호출하면 에러가 나요.

### data가 undefined인 이유

- fetchrk 비동기이기 때문임

```ts
컴포넌트 렌더링 시작
    ↓
useQuery 실행 → fetch 시작 (비동기, 기다리지 않음)
    ↓
data = undefined (아직 응답 안 옴)
    ↓
treatments.map(undefined) → 💥 에러
    ↓
... 잠시 후 ...
    ↓
fetch 완료 → data = [...] → 리렌더링
```

---

## 해결 방법 비교

### 방법 1 - early return (이전 방식)

```ts
if (isLoading) return <div>Loading...</div>
// data가 undefined인 상태에서 렌더링 자체를 막음
```

### 방법 2 - fallback (이 프로젝트 방식)

```ts
const fallback: Treatment[] = []

const { data = fallback } = useQuery({...})
//           ↑
// data가 undefined면 빈 배열을 기본값으로 사용
```
