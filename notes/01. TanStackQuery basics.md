# TanStack Query: `useQuery` 기본 사용법

## 기본 구조

```tsx
const { data, isError, isLoading } = useQuery({
  queryKey: ["posts"],
  queryFn: fetchPosts,
});
```

---

## 필수 옵션 2가지 (순서 상관없음)

### 1. `queryKey` — 쿼리 캐시의 식별자

- **항상 배열** 형태로 작성 (v4 이상 필수)
- 캐시 내에서 데이터를 구분하는 고유 키
- 같은 키 = 같은 캐시 데이터 공유

```tsx
queryKey: ["posts"]; // 단순 키
queryKey: ["posts", postId]; // 동적 키 (ID에 따라 다른 캐시)
queryKey: ["posts", { page }]; // 객체 포함 가능
```

### 2. `queryFn` — 실제 데이터를 가져오는 함수

- 비동기 함수 (Promise를 반환해야 함)
- axios, fetch, GraphQL 호출 등 모두 가능

```tsx
queryFn: fetchPosts; // 함수 참조
queryFn: () => fetchPosts(); // 익명 함수로도 가능
```

---

## 반환값 주요 상태

| 상태         | 설명                                      |
| ------------ | ----------------------------------------- |
| `data`       | 쿼리 성공 시 반환된 데이터                |
| `isLoading`  | 캐시 없음 + 최초 로딩 중                  |
| `isFetching` | 현재 비동기 요청 진행 중 (캐시 유무 무관) |
| `isError`    | 쿼리 실패 시 `true`                       |
| `error`      | 에러 객체                                 |

---

## 상태에 따른 조기 반환 패턴

```tsx
const { data, isError, error, isLoading } = useQuery({
  queryKey: ["posts"],
  queryFn: fetchPosts,
});

// 1. 로딩 중 (보여줄 캐시 데이터 없음)
if (isLoading) return <h3>Loading...</h3>;

// 2. 에러 발생
if (isError) return <h3>에러입니다</h3>;

// 3. 정상 렌더링 (data 안전하게 사용 가능)
return (
  <ul>
    {data.map((post) => (
      <li key={post.id}>{post.title}</li>
    ))}
  </ul>
);
```

> `isLoading` → `isError` 순서로 체크하는 것이 일반적인 패턴

## 에러 처리

```tsx
if (isError) return <h3>{error.toString()}</h3>;
```

- `error` 객체를 통해 에러 메시지를 직접 표시할 수 있음
- `throw new Error('message')` 로 던진 커스텀 에러 메시지도 `.toString()`으로 출력 가능

> **자동 재시도**: TanStack Query는 데이터 fetching 실패 시 기본적으로 **3번 재시도** 후 최종 실패로 판단함

---

## 전체 예시 코드

```tsx
import { useQuery } from "@tanstack/react-query";
import { fetchPosts } from "./api";

export function Posts() {
  const { data, isError, isLoading } = useQuery({
    queryKey: ["posts"],
    queryFn: fetchPosts,
  });

  if (isLoading) return <h3>Loading...</h3>;
  if (isError) return <h3>Error!</h3>;

  return (
    <ul>
      {data.map((post) => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```

## 참고: `isFetching` vs `isLoading`

|                                  | `isFetching` | `isLoading` |
| -------------------------------- | :----------: | :---------: |
| 최초 로드 중 (캐시 없음)         |      ✅      |     ✅      |
| 백그라운드 리패치 중 (캐시 있음) |      ✅      |     ❌      |

- **`isLoading`** → 전체 로딩 화면 (데이터가 아예 없을 때)
- **`isFetching`** → 상단 인디케이터 (백그라운드 갱신 표시)
