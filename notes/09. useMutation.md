# TanStack Query: `useMutation`

## Mutation이란?

서버의 데이터를 **실제로 변경**하는 네트워크 요청.

| 작업                 |     해당 여부      |
| -------------------- | :----------------: |
| 데이터 조회 (Read)   |  ❌ → `useQuery`   |
| 데이터 생성 (Create) | ✅ → `useMutation` |
| 데이터 수정 (Update) | ✅ → `useMutation` |
| 데이터 삭제 (Delete) | ✅ → `useMutation` |

---

## useQuery와의 차이점

|                | `useQuery`   | `useMutation`                     |
| -------------- | ------------ | --------------------------------- |
| 목적           | 데이터 조회  | 데이터 변경                       |
| Query Key      | ✅ 필요      | ❌ 불필요                         |
| 캐시           | ✅ 있음      | ❌ 없음                           |
| `isFetching`   | ✅ 있음      | ❌ 없음 (캐시 없으므로 의미 없음) |
| `isLoading`    | ✅ 있음      | ✅ 있음                           |
| 실패 시 재시도 | 기본 **3회** | 기본 **0회** (설정 가능)          |

> Query Key가 없는 이유: 서버를 변경하는 요청이므로 결과를 캐시할 필요가 없음

---

## 기본 사용법

```tsx
import { useMutation } from "@tanstack/react-query";

const { mutate, isLoading, isError } = useMutation({
  mutationFn: (postId) => deletePost(postId),
});

// 실제 호출
<button onClick={() => mutate(post.id)}>Delete</button>;
```

- `useMutation`은 **`mutate` 함수**를 반환
- `mutate`를 호출하는 시점에 서버로 요청이 날아감

---

## Mutation 이후 화면 반영 방법 (예고)

Mutation 자체는 서버 데이터를 바꾸지만, **React Query 캐시는 자동으로 갱신되지 않음.**  
아래 3가지 방법으로 직접 처리해야 함.

| 방법                   | 설명                                              |
| ---------------------- | ------------------------------------------------- |
| **Optimistic Update**  | 서버 응답 전에 미리 UI 업데이트, 실패 시 롤백     |
| **캐시 직접 업데이트** | 서버 응답 데이터로 React Query 캐시를 직접 수정   |
| **쿼리 무효화**        | `invalidateQueries`로 캐시를 무효화 → 자동 리패치 |

→ 실전 적용은 Day Spa 앱에서 다룰 예정
