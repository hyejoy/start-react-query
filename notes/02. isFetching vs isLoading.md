# TanStack Query: `isFetching` vs `isLoading`

## 한 줄 요약

> **`isLoading`** = 캐시 없음 + 로딩 중  
> **`isFetching`** = (캐시 유무와 관계없이) 현재 비동기 요청 진행 중

---

## 개념 정리

### `isFetching`

- 비동기 쿼리(axios, GraphQL 등)가 **현재 실행 중**임을 의미
- 캐시 데이터가 있든 없든 상관없이, 백그라운드에서 fetch가 일어나고 있으면 `true`
- 즉, **"지금 서버에 요청 날리는 중이야"** 의 신호

### `isLoading`

- 쿼리가 **처음 실행 중**이고, **캐시된 데이터도 없는** 상태
- `isFetching === true` + `캐시 없음` 의 조합
- 즉, **"보여줄 데이터가 아예 없어"** 의 신호

---

## 차이점 비교

| 상태                             | `isFetching` | `isLoading` |
| -------------------------------- | :----------: | :---------: |
| 최초 데이터 로드 중 (캐시 없음)  |  ✅ `true`   |  ✅ `true`  |
| 백그라운드 리패치 중 (캐시 있음) |  ✅ `true`   | ❌ `false`  |
| 요청 없음 (캐시 있음, idle)      |  ❌ `false`  | ❌ `false`  |

---

## 언제 뭘 써야 할까?

```tsx
const { data, isFetching, isLoading } = useQuery({
  queryKey: ["posts"],
  queryFn: fetchPosts,
});

// 아직 보여줄 데이터가 없을 때만 로딩 스피너 표시
if (isLoading) return <LoadingSpinner />;

return (
  <>
    {/* 백그라운드 리패치 중일 때 상단에 인디케이터 표시 */}
    {isFetching && <div>업데이트 중...</div>}
    <PostList data={data} />
  </>
);
```

### 정리

- **`isLoading`** → 스켈레톤 UI / 전체 로딩 화면에 사용 (데이터가 아예 없을 때)
- **`isFetching`** → 상단 로딩 바, 새로고침 인디케이터에 사용 (백그라운드 갱신 표시)

---

## 참고

- `isLoading`은 내부적으로 `isFetching && isPending`과 동일
- TanStack Query v5부터 `isInitialLoading`이 `isLoading`으로 통합됨
