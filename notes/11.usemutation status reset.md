# TanStack Query: `useMutation` 상태 표시 & reset

## useMutation 반환 상태값

`useMutation`은 캐시가 없기 때문에 `useQuery`보다 반환 상태가 단순함.

| 상태        | 설명         |
| ----------- | ------------ |
| `isPending` | 요청 진행 중 |
| `isError`   | 요청 실패    |
| `isSuccess` | 요청 성공    |
| `error`     | 에러 객체    |

> **`isLoading` / `isFetching` 구분 없음**  
> 캐시가 없으므로 둘을 구분할 의미가 없음 → `isPending` 하나로 통합

> **`isSuccess`를 쓰는 이유**  
> `useQuery`는 데이터 자체가 성공의 증거라 `isSuccess`가 필요 없었지만,  
> mutation은 서버 변경이 목적이라 반환 데이터가 없음 → `isSuccess`로 성공 여부 판단

---

## PostDetail에서 상태 표시

```tsx
export function PostDetail({ post, deleteMutation }) {
  return (
    <>
      <button onClick={() => deleteMutation.mutate(post.id)}>Delete</button>

      {deleteMutation.isPending && (
        <p className="loading">Deleting the post...</p>
      )}
      {deleteMutation.isError && (
        <p className="error">
          Error deleting the post: {deleteMutation.error.toString()}
        </p>
      )}
      {deleteMutation.isSuccess && <p className="success">Post was deleted!</p>}
    </>
  );
}
```

---

## 문제: 다른 포스트 클릭 시 이전 mutation 상태가 남음

```
1번 포스트 → Delete 클릭 → isSuccess = true → "Post was deleted!" 표시
2번 포스트 클릭
→ deleteMutation이 Posts에서 선언됐기 때문에 상태가 유지됨
→ 2번 포스트에도 "Post was deleted!" 가 그대로 표시됨 ❌
```

---

## 해결: 포스트 선택 시 mutation reset

포스트 제목 클릭 시 `setSelectedPost`와 함께 `deleteMutation.reset()` 호출.

```tsx
// Posts.jsx
<li
  onClick={() => {
    setSelectedPost(post);
    deleteMutation.reset(); // 이전 mutation 상태 초기화
  }}
>
  {post.title}
</li>
```

`reset()`은 `isPending` / `isError` / `isSuccess` 등 **모든 상태를 초기값으로 되돌림.**

---

## reset이 필요한 이유 정리

```
deleteMutation은 Posts에 선언 → 포스트가 바뀌어도 상태 유지 (언마운트 안 됨)
→ 다른 포스트 선택 시 이전 상태가 그대로 보임

→ 포스트 클릭 시점에 reset() 호출로 상태 초기화 필수
```

> 만약 `PostDetail` 안에서 `useMutation`을 선언했다면?  
> 포스트가 바뀔 때 컴포넌트가 리마운트되면서 자동 초기화됨.  
> 하지만 mutation 상태가 불안정해지는 문제가 생겨서 `Posts`에 두는 것이 올바른 패턴.  
> → **trade-off**: 상태 안정성을 위해 `Posts`에 선언, 대신 `reset()`으로 수동 초기화
